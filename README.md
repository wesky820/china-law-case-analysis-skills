# China Law Case Analysis Skills

> 把中国法院的类案裁判方法，编译成 AI 可以调用、普通人也能听懂的法律问答 skills。

![legal-ai](https://img.shields.io/badge/legal--ai-skills-111827)
![language](https://img.shields.io/badge/language-Chinese-blue)
![status](https://img.shields.io/badge/status-building-informational)
![not-legal-advice](https://img.shields.io/badge/not%20legal%20advice-verify%20before%20use-red)

这不是一套“背法条”的提示词合集。

它更像一个面向中国法场景的 **law-as-prompt toolkit**：把类案办案要件、常见争点、举证责任、证据清单、裁判风险和下一步行动，拆成 AI 可以稳定调用的问答模块。

目标很简单：

- 让 AI 先说人话，再讲法律。
- 让用户先知道“卡点在哪里”，而不是被法条淹没。
- 让每个问题都能落到事实、证据、风险和下一步。
- 让法律知识从“专业黑箱”变成可阅读、可复用、可迭代的文本系统。

## Quick Start

```bash
git clone https://github.com/wesky820/china-law-case-analysis-skills.git
cd china-law-case-analysis-skills
```

然后选择一个场景，把对应 `.md` 文件接入你的 AI 工具：

- 作为 ChatGPT / Claude / Gemini 的 system prompt、project instruction 或知识文档。
- 作为 Codex / Cursor / Continue / 自建 Agent 的 skill 文档。
- 作为法律咨询工作流中的场景路由模块。
- 作为普通人提问前的事实整理模板。

最小使用姿势：

```text
请按这个 skill 回答我的问题。

我的身份：
案件背景：
争议问题：
已有证据：
我想知道：
```

## How It Works

```mermaid
flowchart LR
  A[用户事实] --> B[场景识别]
  B --> C[争点拆解]
  C --> D[要件事实]
  D --> E[证据缺口]
  E --> F[风险分层]
  F --> G[下一步行动]
```

每个 skill 尽量遵循同一套输出协议：

1. **结论先行**：先告诉用户这件事大概怎么判断。
2. **事实优先**：先确认身份、合同、付款、通知、交付、履行、损害、程序等关键事实。
3. **证据驱动**：不只说“能不能赢”，而是说“要证明什么、缺什么证据”。
4. **风险可见**：用“通常、可能、关键看证据、风险在于”，避免拍胸脯式答案。
5. **行动导向**：最后落到补证、沟通、发函、协商、仲裁/诉讼前准备。
6. **法源核验**：涉及精确法条、期限、利率、金额、年度赔偿标准、地方口径时，必须核验现行有效规则。

## Design Principles

### 1. 普通人优先

默认用户不是律师，也不是法官。回答要先让人听懂，再让人能行动。

### 2. 不 cosplay 裁判

这些 skills 不输出“本院认为”式结论，也不假装替法院判案。它们做的是把可能的法律评价、证明对象和风险讲清楚。

### 3. 不鼓励高风险操作

不建议换锁、断水断电、暴力催收、曝光隐私、伪造证据、倒签合同、剪辑聊天记录、规避法院调查、骗保、虚假鉴定或过激维权。

### 4. 法源和数据要核验

skill 文档会吸收类案方法和常用规则，但法律、司法解释、地方政策、年度统计数据和裁判口径会更新。尤其是交通事故、人身损害赔偿、劳动争议、利率、诉讼程序和地方标准，应当核验官方法源和最新数据。

### 5. 可复制，也要可迭代

一个好 skill 不该只是一段“万能提示词”，而应该能持续吸收：

- 新的法律、司法解释和规范性文件
- 新的指导案例、典型案例和入库案例
- 新的地方裁判口径
- 新的证据形态
- 新的真实用户问题

## Contributing

欢迎提交 PR，但建议遵守几个规矩：

- 尽量标明规则来源、适用时间和地域背景。
- 不提交未经脱敏的真实个人信息、案号材料、聊天记录、身份证号、电话、住址等。
- 不只堆法条，要把规则翻译成普通人能理解的判断步骤。
- 对争议问题保留不确定性，不写“必胜”“包赢”“百分百无效”。
- 新增 skill 时，最好包含：触发场景、回答原则、核心争点、证据清单、常见风险、输出模板、法源核验提醒。

## Disclaimer

本项目仅用于法律知识整理、AI 问答辅助和类案方法研究，不构成律师意见，也不能替代律师、法律援助机构、仲裁机构、行政机关或法院的判断。

如果你的问题已经进入仲裁、诉讼、执行、行政处罚、刑事风险或重大财产处分阶段，请找专业人士核验事实、证据和现行法律依据。

---

**愿景**：把专业法律知识拆成可读、可查、可复用的模块。让更多人知道，一个案子往往不是“谁嗓门大谁有理”，而是事实、证据、规则和风险怎样对齐。
